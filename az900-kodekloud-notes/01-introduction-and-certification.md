# Module 1 — Introduction and Certification

## What AZ-900 covers

AZ-900 is a fundamentals certification. It tests whether you understand:

- General cloud concepts
- Core Azure architecture and services
- Azure management, governance, cost, and monitoring tools

The course groups the exam into three broad areas:

| Area | Course weighting |
|---|---:|
| Describe cloud concepts | 25–30% |
| Describe Azure architecture and services | 35–40% |
| Describe Azure management and governance | 30–35% |

Exam objectives and percentages can change, so use the latest official Microsoft study guide for final preparation.

## Exam mindset

- Expect conceptual and scenario-based questions, not deep implementation tasks.
- Learn the purpose of each service and how it differs from similar services.
- Watch for words such as **managed**, **serverless**, **private connection**, **least privilege**, and **high availability**.
- Eliminate answers that provide more complexity or control than the scenario needs.

## Core map

| Need | Azure concept or service |
|---|---|
| Rent computing resources | Cloud computing |
| Create isolated networks | Azure Virtual Network |
| Run a managed web application | Azure App Service |
| Store unstructured objects | Azure Blob Storage |
| Manage identities | Microsoft Entra ID |
| Control who can perform actions | Azure RBAC |
| Enforce configuration rules | Azure Policy |
| Estimate a future Azure bill | Pricing Calculator |
| See personalized optimization advice | Azure Advisor |

## Quick check

1. Is AZ-900 a deep administrator certification? **No; it validates foundational knowledge.**
2. What should you learn for scenario questions? **The purpose, benefits, and trade-offs of services.**
3. Why check the official study guide? **Exam skills and weightings can change.**
